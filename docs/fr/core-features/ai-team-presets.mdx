Les préréglages d'équipe AI vous permettent d'enregistrer et de réutiliser des configurations de membres AI, au lieu de tout reconfigurer à chaque session.

Il existe deux types de préréglages :

- **Préréglages de membre** - configuration d'un agent AI individuel (rôle, réglage de rôle, type d'agent)
- **Préréglages d'équipe** - groupe de préréglages de membres importable en une seule fois

## Gérer les préréglages

Ouvrez **Paramètres -> Préréglages** pour gérer les modèles de membres et d'équipes.

### Onglet préréglages de membres

- **Ajouter** un préréglage membre personnalisé
- **Modifier** un préréglage personnalisé existant
- **Copier** n'importe quel préréglage comme point de départ
- **Désactiver** un préréglage sans le supprimer
- **Supprimer** les préréglages personnalisés inutiles

### Onglet préréglages d'équipes

- **Ajouter** une équipe en sélectionnant des membres
- **Modifier** les membres d'une équipe personnalisée
- **Copier** un préréglage d'équipe pour créer une variante
- **Désactiver** une équipe pour la masquer de l'import
- **Supprimer** les équipes personnalisées

## Utiliser les préréglages dans une session

### Ajouter un préréglage de membre

Dans la **barre latérale AI Members** à droite du chat :

1. Cliquez sur **Ajout rapide de membres** au-dessus de la liste des membres
2. Parcourez les préréglages disponibles
3. Cliquez sur un préréglage pour l'ajouter instantanément à la session

Le nouveau membre est créé avec son nom, son type d'agent et son réglage de rôle. Vous pouvez encore modifier ensuite le chemin de workspace, l'exécuteur, etc.

### Importer un préréglage d'équipe

1. Cliquez sur **Import Team** dans la barre AI Members
2. Sélectionnez le préréglage d'équipe à importer
3. Vérifiez la **prévisualisation d'import** :
   - **Created** - un nouveau membre sera créé
   - **Reused** - un membre identique existe déjà
   - **Auto-renamed** - conflit de nom avec config différente, suffixe `_2` (ou plus)
4. Confirmez l'import

<Tip>
La prévisualisation permet d'anticiper les conflits de nom. Si un membre du même nom existe avec une configuration différente, le nouveau sera renommé automatiquement.
</Tip>

### Règles de résolution des conflits

| Scénario | Action |
|---|---|
| Même nom + même config | Réutiliser le membre existant |
| Même nom + config différente | Créer un nouveau membre avec suffixe `_2` (ou `_3`, etc.) |
| Membre déjà présent dans la session | Conserver le membre existant |

## Sessions archivées

Dans une session archivée, **les contrôles d'import sont désactivés**. Vous ne pouvez pas ajouter de membres ni importer d'équipes.
